# Relatório de Correção de Tipagem Nominal - DiskStorage
Data: 01-02-2026_07-26

## O que foi feito
1.  **Casting de Tipos Nominais**: Foram aplicadas as funções `asFileName` e `asBytes` em `src/storage/disk.ts` para converter tipos primitivos (`string`, `number`) nos tipos branded (`FileName`, `Bytes`) exigidos pela interface `File`.
2.  **Padronização de Callbacks**: As assinaturas dos métodos internos `getFilename` e `defaultFilename` foram atualizadas para utilizar `FileName` ao invés de `string` pura nos parâmetros de callback.
3.  **Atualização de Opções**: A interface `DiskStorageOptions` em `src/types.ts` foi modificada para aceitar `FileName` em suas propriedades e callbacks, mantendo a consistência com o sistema de tipos nominais do projeto.
4.  **Correção de Lógica**: O método `getFilename` no `DiskStorage` foi corrigido para utilizar o valor da string de `this.filename` quando fornecido pelo usuário, ao invés de sempre cair no fallback do `defaultFilename`.

## Por que foi feito
A mudança foi necessária para resolver erros de compilação do TypeScript que impediam o build do projeto. Além disso, a refatoração segue a diretriz técnica de utilizar **Tipagem Semântica Nominal** para evitar confusão entre tipos de dados primitivos que possuem significados diferentes na aplicação (como o tamanho em bytes e o nome do arquivo).

A correção da lógica de `getFilename` foi identificada durante a análise do código, garantindo que as opções de configuração passadas pelo usuário sejam devidamente respeitadas pelo mecanismo de storage em disco.
