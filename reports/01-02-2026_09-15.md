# Relatório de Compatibilidade TypeScript ESM
Data: 01-02-2026_09-15

## Problema
Ocorreram erros de compilação TS5097 ("An import path can only end with a '.ts' extension when 'allowImportingTsExtensions' is enabled") ao tentar utilizar imports com extensão `.ts` nos arquivos fonte.

## Solução Técnica Adotada
Reversão para imports com extensão `.js` (e.g., `import ... from "./types.js"`).

## Justificativa Técnica
Embora pareça contra-intuitivo para um projeto TypeScript, o uso de `.js` nos imports é **obrigatório** para projetos que compilam via `tsc` (TypeScript Compiler) com configuração de saída ESM (`"module": "node16"` ou `"nodenext"`).

1.  **Geração de .d.ts**: O `tsc` não reescreve as extensões dos imports nos arquivos de definição de tipos (`.d.ts`). Se usarmos `.ts` no código fonte, os arquivos `.d.ts` distribuídos conterão imports para arquivos `.ts` inexistentes no pacote final, quebrando o uso da biblioteca por terceiros.
2.  **Padrão Node ESM**: O Node.js e o ecossistema ESM exigem extensões explícitas. O TypeScript se alinha a isso exigindo que o desenvolvedor escreva a extensão que o arquivo terá *após* a compilação (ou seja, `.js`).
3.  **Bun**: O Bun é capaz de executar/resolver ambos, mas manter a compatibilidade com `tsc` é vital para a publicação do pacote no NPM.
