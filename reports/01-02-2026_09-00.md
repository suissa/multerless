# Relatório de Inconsistência - Node Experimental TS
Data: 01-02-2026_09-00

## Problema Identificado
O comando `node --experimental-strip-types src/cac/executor.ts` falha com `ERR_MODULE_NOT_FOUND` porque o resolver nativo do Node.js (mesmo em versões experimentais) não realiza o mapeamento de extensões `.js` para arquivos `.ts` em tempo de execução para imports ESM.

## Análise Técnica
1.  **Compatibilidade TSC**: O projeto utiliza `tsc` para gerar saída ESM, o que exige que os imports contenham a extensão final `.js`.
2.  **Limitação do Node**: O flag `--experimental-strip-types` do Node.js é projetado para arquivos isolados ou projetos onde os imports são exatos. Ele não possui a inteligência de resolução de módulos que o Bun ou loaders como `tsx` possuem.
3.  **Regra de Ouro**: O projeto possui a regra **"SEMPRE utilize bun em detrimento ao node"**, justamente para evitar essas fragilidades de flags experimentais do Node.

## Solução Recomendada
Para manter a compatibilidade com o build oficial e garantir a execução estável:
- **Recomendado (Bun)**: `wsl bun src/cac/executor.ts` (ou `wsl bun cac`).
- **Alternativa (Node no Build)**: `wsl node dist/cac/executor.js` (após o build).

## Ações Tomadas
As tentativas de forçar a compatibilidade via `allowImportingTsExtensions` foram revertidas, pois causavam conflitos com a emissão de código do `tsc`, que é vital para a distribuição da biblioteca.
