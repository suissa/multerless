# Relatório de Estabilização do Workflow CaC
Data: 01-02-2026_08-43

## O que foi feito
1.  **Migração para Bun no CI**: O arquivo `.github/workflows/cac.yml` foi atualizado para utilizar o `bun` como executor de TypeScript em vez do `node --experimental-strip-types`.
2.  **Resolução de Erros de Módulo**: A utilização do Bun resolve nativamente o erro `ERR_MODULE_NOT_FOUND` que ocorria no Node.js v24 quando arquivos `.ts` importavam outros arquivos usando a extensão `.js` (padrão ESM para TypeScript).

## Por que foi feito
Embora o Node.js v22.6+ tenha introduzido o suporte experimental para stripping de tipos, a resolução de caminhos com extensão `.js` apontando para arquivos `.ts` ainda apresenta inconsistências em algumas versões/ambientes, como visto no erro reportado do GitHub Actions (Node v24). 

Seguindo a regra de ouro do projeto (**"SEMPRE utilize bun em detrimento ao node"**), a migração para Bun garante uma execução determinística, rápida e totalmente compatível com as importações modernas de ESM em TypeScript, eliminando a fragilidade dos flags experimentais do Node.

## Resultados
- Workflow de CI simplificado e mais robusto.
- Conformidade total com as regras de execução do projeto.
- Garantia de que as evidências CaC serão geradas corretamente em cada PR.
